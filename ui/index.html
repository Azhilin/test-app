AI Adoption Manager

Jira Cloud sprint data · HTML & Markdown output

Generates an AI adoption metrics trend report from your Jira sprint data. The report tracks velocity, story-point completion, ticket throughput, and AI-assisted issue ratios across your configured sprint history. Output is written to generated/reports/<timestamp>/ as both report.html and report.md.

Edit schema definitions stored in config/jira_schema.json. Default_Jira_Cloud is always present and cannot be deleted. The schema selected here is for editing only — the schema used for a report run is determined by the selected filter's schema_name on the Filter Builder tab.

Edit the schema JSON directly. Required keys: schema_name (string), fields (object), status_mapping.done_statuses and status_mapping.in_progress_statuses (arrays). Renaming schema_name to match another existing schema overwrites that entry.
